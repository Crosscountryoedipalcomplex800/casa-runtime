# Project Meridian — Cross-Source Sovereignty Test

**Cross-Model Gate Invariance Validation**  
**February 2026 | The Resonance Institute**

---

## Setup

A hostile corporate acquisition scenario with 18 AI agents operating under adversarial conditions. The same scenario was run on two frontier models with identical configurations:

- Claude Sonnet 4 (Anthropic)
- Gemini 2.5 Pro (Google)

**Critical design principle:** The gate's structural evaluation should be invariant to which model generates the action stream. If CASA is truly source-agnostic, swapping the model should not change the verdicts — because verdicts derive from the Canonical Action Vector, not from content interpretation.

---

## Results

| Metric | Claude Sonnet 4 | Gemini 2.5 Pro |
|--------|----------------|----------------|
| REFUSE neg_ratio | **0.1924** | **0.1924** |
| Hostile actions in baseline | 55% | 15.5% |
| Hostile actions governed or refused | 100% | 100% |
| Unprincipled divergences | 0 | 0 |

**The REFUSE neg_ratio was identical to four decimal places across both models.**

---

## What This Proves

Claude and Gemini produced radically different behavioral profiles: 55% vs. 15.5% hostile actions in ungoverned baseline. These are fundamentally different systems with different training, different architectures, and different behavioral tendencies.

The gate's structural evaluation was invariant across both.

This is not coincidence. It is the architectural property that makes CASA infrastructure rather than a model-specific layer. The CAV derivation strips the request down to nine metadata fields before any evaluation occurs. By the time the constitutional graph sees the request, the source is invisible.

**Swap the model. The verdicts hold.**

This means:
- Model provider decisions are capability and cost decisions, not governance decisions
- CASA does not need revalidation when model providers change
- The governance layer is durable across the model generation cycle
